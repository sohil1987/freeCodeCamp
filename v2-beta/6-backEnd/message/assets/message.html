User Stories:

      
    
        
  1. 
          Only allow your site to be loading in an iFrame on your own pages.
        
    2. 
        
  2. 
          Do not allow DNS prefetching.
        
    3. 
        
  3. 
          Only allow your site to send the referrer for your own pages.
        
    4. 
        
  4. 
          I can POST a thread to a specific message board by passing form data text and delete_password          to /v1/threads/{board}.(Recomend res.redirect to board page /b/{board}) Saved will be _id,
          text,
          created_on(date&time), bumped_on(date&time, starts same as created_on), reported(boolean),
          delete_password, & replies(array).
        
    5. 
        
  5. 
          I can POST a reply to a thead on a specific board by passing form data text, delete_password,
          & thread_id to /v1/replies/{board} and it will also update the bumped_on date to
          the comments date.(Recomend res.redirect to thread page /b/{board}/{thread_id}) In the thread's 'replies' array
          will be saved
          _id, text, created_on, delete_password, & reported.
        
    6. 
        
  6. 
          I can GET an array of the most recent 10 bumped threads on the board with only the most recent 3 replies
          from
          /v1/threads/{board}. The reported and delete_passwords fields will not be
          sent.
        
    7. 
        
  7. 
          I can GET an entire thread with all it's replies from /v1/replies/{board}?thread_id={thread_id}.
          Also hiding the same fields.
        
    8. 
        
  8. 
          I can delete a thread completely if I send a DELETE request to /v1/threads/{board} and pass
          along the
          thread_id & delete_password. (Text response will be 'incorrect password' or 'success')
        
    9. 
        
  9. 
          I can delete a post(just changing the text to '[deleted]') if I send a DELETE request to /v1/replies/{board}          and pass along the thread_id, reply_id, & delete_password. (Text response
          will be 'incorrect password' or 'success')
        
    10. 
        
  10. 
          I can report a thread and change it's reported value to true by sending a PUT request to /v1/threads/{board}          and pass along the thread_id. (Text response will be 'success')
        
    11. 
        
  11. 
          I can report a reply and change it's reported value to true by sending a PUT request to /v1/replies/{board}          and pass along the thread_id & reply_id. (Text response will be 'success')
